📐 Încadrare Piese WEB Rotary — Ajutor

Programul calculează încadrarea optimă a pieselor pe desfășurata unei ștanțe rotative: piesele se distribuie uniform pe circumferința cilindrului (înaintarea WEB), rezultă lățimea de rolă necesară, iar la final rola se încadrează pe logul de materie primă.

1. Fluxul de lucru recomandat

  • 1. Importă piesa DXF (și opțional piesa B, dacă perechea e din două piese diferite).
  • 2. Setează Înaintarea WEB (circumferința / pasul ștanței rotary), distanța minimă dintre piese și marginea față de marginea benzii.
  • 3. Alege modul (Standard sau Perechi), bifează unghiurile de rotație permise, setează numărul de rânduri și distanța dintre ele.
  • 4. Apasă 🚀 Calculează → rezultă așezarea și lățimea de rolă.
  • 5. La 🪵 Feliere log introdu logul (lățime, lungime, mașină, kerf, capete) și apasă Optimizează felierea → câte role ies din log și câte piese în total.

2. Import DXF

  • Formate suportate: LINE, ARC, LWPOLYLINE, POLYLINE/VERTEX, SPLINE (NURBS cu noduri și grad — exporturile din CorelDRAW, AutoCAD, Illustrator).
  • Curbele sunt redate fidel (eșantionate fin și simplificate la 0,05 mm). Conturul trebuie să fie închis, în secțiunea ENTITIES.
  • Piesa B diferită (opțional): în modul Perechi, perechea devine A + B (raport mereu 1:1, ex. capac + bază). Se elimină cu ✕.

3. Noțiuni rotary

  • Înaintare WEB = circumferința cilindrului. Webul afișat este desfășurata lui: capătul din dreapta se continuă cu cel din stânga (sutura).
  • Piesele se așază cu pas uniform = înaintare / N → toate distanțele de-a lungul webului sunt identice, inclusiv la sutură; la capetele desenului rămâne câte jumătate din distanță.
  • Piesele care trec peste capăt se desenează înfășurat (tăiate și continuate pe partea opusă) — exact ca pe cilindru.
  • Toate distanțele sunt măsurate real, între contururi (euclidian), nu între dreptunghiuri de încadrare.

4. Modul Standard

Toate piesele au aceeași orientare. Programul încearcă unghiurile bifate la „Rotații" și alege unghiul care dă cele mai multe piese pe lățimea de rolă. Distribuție uniformă pe înaintare, rânduri la distanța setată, lățimea de rolă rezultă din calcul.

5. Modul Perechi

  • Perechea = piesa A + piesa B rotită cu 180° (sau piesa B diferită), imbricate optim una în alta.
  • Distanțele se egalizează: d în pereche = d între perechi = d la sutură (toate ≥ limita setată) — necesar pentru ștanțe cu pas uniform.
  • Rotațiile bifate se aplică perechii (A la unghi θ, B la θ+180°).

6. Rânduri

Numărul de rânduri multiplică rândul optimizat pe lățimea benzii. Distanța dintre rânduri o setezi tu (măsurată real, cu verificare și pe diagonală între rândurile vecine). Rândurile se întrepătrund automat unde forma permite. Lățime rolă = 2×margine + înălțimea rândului/perechii + (rânduri−1)×pas Y.

7. Feliere log

  • Logul (1000–1300 mm, 33–500 m) se taie în role identice de lățimea rezultată.
  • Parametri: lățimea/lungimea logului, lățimea max. a mașinii (250/350 mm), kerf (pierderea lamei), pierderile la capete.
  • Rezultat: numărul de role, restul de lățime, utilizarea %, piese/m și piese pe tot logul, cu bară vizuală a tăierii.
⚠ Dacă rola depășește lățimea mașinii, apare avertisment roșu — redu numărul de rânduri.

8. Modul Manual

  • ➕ Piesa A / ➕ Piesa B = plasare cu click (verde = valid, roșu = coliziune); ↖ Sel = selectare/mutare prin tragere.
  • A/B/S schimbă unealta, Q/E rotește ±5°, ←/→ ±1°, Enter plasează, Del șterge selecția, Ctrl+Z anulează, Esc deselectează, click dreapta șterge piesa.
  • Ctrl+Click = selecție multiplă; Copiază selecția multiplică pe X cu pasul dat; ⟹ Umple WEB repetă tot grupul pe lungime.

9. Citirea rezultatelor

  • Pas X × N — pasul uniform și câte piese/perechi pe o rotație de cilindru.
  • d intra / d inter / sutură — distanțele reale: în pereche / între perechi / la închiderea cilindrului.
  • Capete — golul la fiecare capăt al desenului (½ din distanță) sau „wrap" dacă piesele se întrepătrund peste sutură.
  • Verificare min. — distanța minimă măsurată în toată așezarea: verde = respectă limita, roșu = nu (semnalează o problemă).

10. Sfaturi

  • Mai multe unghiuri bifate = rezultat mai bun, dar calcul mai lent (~1–2 s/unghi la Perechi).
  • Mărește numărul de rânduri până când rola se apropie de lățimea mașinii — utilizare maximă a logului.
  • Navigare: rotița = zoom, tragere = panoramare, Fit = încadrare în ecran, 💾 PNG = export imagine.
